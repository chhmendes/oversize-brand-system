# Grafismos

## 1. O Papel dos Grafismos

Na Oversize, o grafismo não é decoração. É **estrutura visível**.

Todo elemento gráfico deve ter função. Se cobrir o logo e a marca ainda é reconhecível pelas cores, formas e composição, os grafismos estão funcionando.

O teste é direto:

> **Se eu cobrir o logotipo, a página ainda parece Oversize?**

---

## 2. A Linguagem Visual

### Flat + Poster

A Oversize é uma marca **gráfica e flat**. Sem gradientes, sem blur, sem material design, sem sombras difusas.

A profundidade vem de **planos sólidos sobrepostos**, não de efeitos.

Pense em cartaz tipográfico, não em interface moderna. Pense em pôster de impacto, não em aplicativo SaaS.

### Formas: geometria sharp

Cantos quadrados ou com radius mínimo (`--r-0` a `--r-3`, máximo 8px para cards).

Sem arredondamentos excessivos. Sem blobs. Sem formas orgânicas.

Formas permitidas:
- retângulos com cantos sharp ou mínimos;
- linhas retas (1–2px);
- grades e grids;
- blocos de cor sólida;
- badges/tags em pill (`--r-pill`) apenas para elementos muito pequenos (etiquetas, avatares).

---

## 3. O Shadow Hard — Assinatura Visual

O elemento mais proprietário da identidade Oversize é o **shadow hard** (sombra offset):

```
box-shadow: 6px 6px 0 0 #222222;  /* --sh-hard */
box-shadow: 6px 6px 0 0 #B51F3A;  /* --sh-hard-red */
```

É uma sombra sem blur — deslocamento puro, como em cartazes impressos e zines.

### Quando usar

- Botão primário com `hardShadow` prop
- Cards de destaque
- Caixas de callout
- Elementos hero que precisam de "peso"

### Quando não usar

- Todos os cards de uma grade (use com parcimônia)
- Texto
- Imagens
- Navigation items

### A versão vermelha

`--sh-hard-red` (6px 6px 0 0 #B51F3A) aparece sobre fundos escuros (graphite, graphite-deep) onde o shadow dark seria invisível.

---

## 4. Bordas

As bordas da Oversize são **2px** em grafite ou vermelho. Nunca 1px (muito tímido para a postura da marca).

| Contexto | Borda |
|---|---|
| Card padrão | 2px solid `#222222` |
| Card de destaque / hover | 2px solid `#B51F3A` |
| Input em foco | 1px solid `#B51F3A` + ring 3px `#F5E3E7` |
| Dividers | 1px solid `#D9D9D9` |
| Blockquote | 3px solid `#B51F3A` (borda esquerda) |

---

## 5. Cards — O Componente Base

O card é o bloco construtivo da identidade.

**Anatomia padrão:**

```
border-radius: 8px  (--r-3)
border: 2px solid #222222
box-shadow: 6px 6px 0 0 #222222  (--sh-hard)
background: #FFFFFF
padding: 20px
margin-right: 6px  (compensa o shadow)
margin-bottom: 6px
```

**Card accent (fundo vermelho):**

```
background: #B51F3A
border: 2px solid #8E1A2E
box-shadow: 6px 6px 0 0 #8E1A2E
text: #D9D9D9 (ash)
```

**Hover em cards linkáveis:**

```
border-color: #B51F3A
box-shadow: 6px 6px 0 0 #B51F3A
```

---

## 6. Superfícies e Backgrounds

A marca usa **quatro fundos principais**:

| Fundo | Hex | Atmosfera |
|---|---|---|
| Branco `--bg-1` | `#FFFFFF` | Clean, editorial |
| Ash-soft `--bg-2` | `#FAFAFA` | Painel sutil |
| Graphite `--bg-dark-1` | `#3C3C3C` | Dark, presença |
| Red `--bg-accent` | `#B51F3A` | Hero, urgência |

Transições entre seções devem usar esses quatro. Nunca fundos intermediários que enfraqueçam o contraste.

---

## 7. Label Tipográfico como Grafismo

Um dos elementos proprietários mais usados é o **label de seção**:

```
font-size: 10px
font-weight: 700
letter-spacing: 0.12em
text-transform: uppercase
color: --ov-red
```

Aparece como: `BRAND` · `01 / CORES` · `COLORS` · `TYPE`

É sempre em vermelho. É sempre em caixa alta. É sempre pequeno.

Esse label cria consistência visual imediata: onde quer que apareça, é Oversize.

---

## 8. Grafismos Proibidos

A Oversize não usa:

- **Gradientes** — em nenhuma superfície ou elemento
- **Sombras difusas** como principal elemento de profundidade
- **Blur / frosted glass** — não existe na identidade
- **Blobs e formas orgânicas** — a marca é geométrica
- **Ícones filled/preenchidos** — apenas stroke (Lucide, 2px stroke)
- **Ilustrações figurativas** — a marca usa grafismos, não ilustração
- **Backgrounds com textura ou padrão repetitivo** (em v1)
- **Animações em loop ambiente** — nada que pulsa ou shimmer

---

## 9. Iconografia

A Oversize usa **Lucide** como biblioteca de ícones.

**Regras:**
- Stroke-only (nunca filled)
- Stroke-width: `2` (matching peso bold do texto ao redor)
- Tamanho: 16–20px para inline com corpo; 24–32px standalone
- Cor: `currentColor` (herda da cor do texto)
- Vermelho apenas em ícones de ação isolados ou status

---

## 10. Composição

### Alinhamento

Hard left. O wordmark é left-aligned. As composições da Oversize preferem alinhamento à esquerda.

Centrado apenas para elementos isolados (hero, logotipo standalone).

### Espaçamento

Generoso. O espaço em branco não é "espaço vazio" — é sinal de clareza e controle.

Usar `--sp-7` (48px) como margens de layout em desktop, `--sp-4` (16px) em mobile.

### Grid

12 colunas em desktop, gutter `24px` (`--sp-5`), margem `48px` (`--sp-7`).

---

## 11. Teste de Reconhecimento

A identidade visual está funcionando quando:

1. A página tem exatamente um elemento vermelho principal
2. As bordas e shadows têm presença (não parecem "delicadas")
3. A tipografia está em caixa baixa nos títulos
4. Os fundos são sólidos (sem gradiente)
5. A composição parece um pôster — não um template

---

## 12. Síntese

Os grafismos da Oversize nascem de uma decisão simples: **ser flat, ser bold, ser claro**.

Shadow hard. Borda 2px. Três cores. Formas geométricas.

> "Grafismo não é enfeite. É onde a identidade continua depois que o logo acaba."
