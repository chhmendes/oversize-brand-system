# Aplicações

## 1. O Princípio dos Touchpoints

> **A marca não precisa estar em todos os lugares. Precisa ser inconfundível nos lugares certos.**

Antes de criar uma aplicação, uma pergunta:

**O público-alvo da Oversize está lá, neste momento, com disposição para receber?**

Se sim, a marca vai. Com todo o sistema.

---

## 2. Prioridade dos Pontos de Contato

A Oversize concentra energia em:

| Prioridade | Canal | Por quê |
|---|---|---|
| 1 | Brand System (este site) | Base do sistema. Onde tudo começa. |
| 2 | LinkedIn | Onde o ICP passa mais tempo |
| 3 | Proposta comercial | Momento de decisão de compra |
| 4 | Apresentações (Keynote/Slides) | Principal ferramenta de venda e diagnóstico |
| 5 | Papelaria digital | Cartão de visitas, assinatura de e-mail |
| 6 | Newsletter | Relacionamento com base qualificada |

---

## 3. Brand System (Este Site)

O próprio Brand System é o touchpoint principal: uma interface onde qualquer designer, copywriter ou parceiro encontra as diretrizes.

### Estrutura do site

- **Sidebar fixa:** navegação por seção (Estratégia, Marca, Comunicação, Identidade)
- **Topbar:** 48px, limpa, apenas logo + botão de acesso
- **Conteúdo:** MDX com tipografia da marca, sem poluição visual
- **Scrollbar:** oculta em toda a interface

### Padrões aplicados

- Fundo branco ou ash-soft como base
- Títulos em Sansa Pro Bold
- Labels de seção em vermelho uppercase
- Cards de navegação com border 2px + sh-hard

---

## 4. LinkedIn

O LinkedIn é onde o ICP da Oversize passa mais tempo. A marca precisa ser reconhecível em 1,5 segundo de scroll.

### Posts tipográficos (prioridade)

- Fundo sólido (branco, grafite ou vermelho)
- Texto grande, bold, lowercase
- Label de seção no canto superior
- Zero decoração

**Dimensões:** 1080×1080px

### Carrosséis estruturados

- Capa com fundo graphite ou vermelho
- Slides internos: branco com tipografia
- Última slide: CTA com botão vermelho

**Regra:** cada slide tem um único ponto. Sem excesso.

### Avatar / perfil

- Foto editorial (não selfie de evento)
- Moldura: círculo com borda vermelha 3px (se disponível na plataforma)

### Banner do perfil

- Fundo graphite
- Wordmark gray-on-dark à esquerda
- Tagline em ash à direita
- Ou: fundo vermelho com wordmark gray-on-red

## 5. Proposta Comercial

A proposta é o documento que transforma uma conversa em decisão.

Visualmente, deve parecer **uma ferramenta de clareza**, não um orçamento de agência.

### Estrutura recomendada

| Seção | Conteúdo |
|---|---|
| Capa | Wordmark + nome do cliente + data |
| Diagnóstico | O problema identificado (com dados) |
| Risco | O custo de não resolver (Diluição Estratégica) |
| Escopo | Entregas estruturadas, sem ambiguidade |
| Método | As etapas (com numeração da Estratégia Estruturada) |
| Investimento | Valor com lógica de ROI |
| Próximos passos | CTA com data |
| Apêndice | Cases e provas |

### Visual

- Formato A4/Letter, orientação vertical
- Fundo branco para texto, capa em graphite
- Tipografia Sansa Pro
- Labels de seção em vermelho uppercase
- Shadow hard em elementos de destaque (boxes de insights)
- Sem foto de banco de imagem

## 6. Apresentações

Keynote/PowerPoint são o palco onde a Oversize apresenta diagnósticos, metodologia e propostas.

### Estrutura de slides

| Tipo de slide | Composição |
|---|---|
| Capa | Fundo graphite ou vermelho, wordmark, título |
| Conceito | Texto display grande, fundo branco |
| Diagrama | Grid com boxes 2px + labels vermelhos |
| Dado | Número grande (Sansa Pro Black) + contexto menor |
| Síntese | Fundo graphite, texto ash, um destaque vermelho |
| CTA | Fundo vermelho, texto ash, botão com sh-hard |

### Dimensões

16:9 (1920×1080px) padrão.

## 7. Papelaria Digital

### Cartão de visitas digital

| Campo | Especificação |
|---|---|
| Frente | Wordmark original + nome + cargo |
| Verso | Graphite + dados de contato em ash |
| Tamanho | 90×50mm / 3.5×2in |

### Assinatura de e-mail

```
[Nome]
[Cargo] · Oversize

[site] · [LinkedIn] · [e-mail]
```

- Sem imagens na assinatura (não aparecem em todos os clientes)
- Nome: Sansa Pro Bold ou substituto do sistema
- Separador: · (ponto centrado) em grafite
- Link: vermelho, sem underline

### Papel timbrado

- Header: logo original (esquerda) + dados de contato (direita)
- Footer: site + linha fina ash
- Corpo: Sansa Pro 11pt, espaço entre linhas 1.4

## 8. Newsletter

A newsletter da Oversize é um ativo editorial — não um disparo promocional.

### Estrutura padrão

| Bloco | Descrição |
|---|---|
| Cabeçalho | Logo + número da edição + data |
| Abertura | Frase conceitual (display estilo) |
| Conteúdo principal | 1 ou 2 tópicos em profundidade |
| Destaque | Box com border 2px grafite |
| CTA | Botão vermelho + texto auxiliar |
| Rodapé | Links de desinscrição + dados legais |

### Tom na newsletter

- Direto, sem enrolação
- Sem emoji
- Sem "clique aqui" (use verbos de ação: "acesse", "leia", "veja")

## 9. Diretrizes de Entrega de Arquivos

Ao entregar assets para produção:

| Formato | Uso |
|---|---|
| `.svg` | Logo para qualquer escala |
| `.png` 2x | Logo para uso digital (fundo transparente) |
| `.pdf` | Documentos e propostas |
| `.otf` | Fontes Sansa Pro (5 pesos) |
| `.key` / `.pptx` | Template de apresentação |

### Organização de pasta de entrega

```
Oversize Brand Assets/
├── logo/
│   ├── original.svg
│   ├── gray-on-dark.svg
│   ├── gray-on-light.svg
│   ├── gray-on-red.svg
│   └── red-on-gray.svg
├── fonts/
│   └── SansaPro-[peso].otf (×5)
├── colors/
│   └── palette.ase (swatches para Adobe)
└── templates/
    ├── proposta.pdf
    └── apresentacao.key
```

## 10. Síntese

As aplicações da Oversize são a prova de que o sistema funciona fora do Brand System.

Cada touchpoint deve responder à mesma pergunta:

> **Parece Oversize mesmo sem o logo?**

Se sim: sistema funcionando.

Se não: revisar antes de publicar.
